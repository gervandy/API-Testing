# Create Case

## Endpoint
POST /api/cases

---

## Headers

| Header | Value |
|---------|-------|
| Authorization | Bearer Token |
| Content-Type | application/json |

---

## Request Body

```json
{
  "patientId": "P10001",
  "patientName": "John Doe",
  "medication": "Aspirin",
  "reaction": "Skin Rash",
  "status": "Open"
}
```

---

## Success Response

Status Code: 201 Created

```json
{
  "caseId": "HCMS-10001",
  "message": "Case created successfully"
}
```

---

## Error Responses

| Code | Description |
|------|-------------|
| 400 | Validation error |
| 401 | Unauthorized |
| 409 | Duplicate case |
| 500 | Server error |

---

## Validation Points

- Status code = 201
- Case ID is generated
- Response schema is correct
- Record is persisted
- Data matches request payload

---

## Test Scenarios

### Positive
- Valid case creation
- Create multiple cases

### Negative
- Missing patientId
- Missing medication
- Invalid auth token
- Duplicate case creation

### Boundary
- Max length patientName
- Special characters in fields
- Large payload request

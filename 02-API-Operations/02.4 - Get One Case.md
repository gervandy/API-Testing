# Get One Case

## Business Purpose
Retrieve a specific healthcare case by case ID.

---

## Endpoint
GET /api/cases/{caseId}

---

## Path Parameters
| Parameter | Type |
|----------|------|
| caseId | String |

Example: HCMS-10001

---

## Success Response
200 OK

```json
{
  "caseId": "HCMS-10001",
  "patientId": "P10001",
  "patientName": "John Doe",
  "medication": "Aspirin",
  "reaction": "Skin Rash",
  "status": "Open"
}
```

---

## Error Responses
- 400 Invalid case ID
- 401 Unauthorized
- 404 Case not found
- 500 Server error

---

## Validation Points
- Status code is correct
- Response matches requested caseId
- All fields are returned correctly
- No data mismatch

---

## Test Scenarios

### Positive
- Retrieve valid case

### Negative
- Invalid case ID format
- Non-existent case ID
- Unauthorized request

### Boundary
- Very long case ID
- Special characters in case ID

# Get Case

## Purpose

Retrieve the details of a specific healthcare case using its unique Case ID.

This endpoint is commonly used to verify that a case has been successfully created or updated before performing further operations.

---

## Endpoint

GET /api/cases/{caseId}

Example

GET /api/cases/HCMS-10001

---

## Request Headers

| Header | Value |
|---------|-------|
| Authorization | Bearer Token |
| Accept | application/json |

---

## Path Parameter

| Parameter | Type | Description |
|-----------|------|-------------|
| caseId | String | Unique identifier of the healthcare case |

Example

HCMS-10001

---

## Success Response

Status Code

200 OK

Example Response

```json
{
    "caseId": "HCMS-10001",
    "patientId": "P10001",
    "patientName": "John Doe",
    "medication": "Aspirin",
    "reaction": "Skin Rash",
    "status": "Open",
    "createdDate": "2026-06-24"
}
```

---

## Validation Points

- Status Code is 200
- Response time is within acceptable limit
- Returned Case ID matches requested Case ID
- Patient information is correct
- Medication information is correct
- Case status is correct
- Response body follows expected JSON structure

---

## Common Error Responses

| Status | Description |
|---------|-------------|
| 400 | Invalid Case ID format |
| 401 | Unauthorized |
| 404 | Case not found |
| 500 | Internal server error |

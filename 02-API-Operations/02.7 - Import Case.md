# Import Case

## Business Purpose
Import multiple healthcare cases using a CSV file.

---

## Endpoint
POST /api/cases/import

---

## Headers
| Header | Value |
|--------|------|
| Authorization | Bearer Token |
| Content-Type | multipart/form-data |

---

## Request Body
- file: CSV file containing case data

Example: cases_import.csv

---

## Success Response
200 OK

```json
{
  "totalRecords": 10,
  "successfulImports": 9,
  "failedImports": 1
}
```

---

## Error Responses
- 400 Invalid file format
- 401 Unauthorized
- 413 File too large
- 415 Unsupported file type
- 500 Server error

---

## Validation Points
- Status code is correct
- File uploads successfully
- Total records match input file
- Failed records are reported correctly
- Valid records are created in system

---

## Test Scenarios

### Positive
- Import valid CSV file

### Negative
- Invalid file type
- Corrupted CSV file
- Missing required columns
- Unauthorized request

### Boundary
- Large file upload
- Maximum row limit
- Empty file

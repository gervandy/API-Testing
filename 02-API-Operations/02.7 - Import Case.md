# Import Case

## Purpose

Import one or more healthcare cases into the system using a CSV file.

This endpoint is commonly used for bulk case creation or migration.

---

## Endpoint

POST /api/cases/import

---

## Request Headers

| Header | Value |
|---------|-------|
| Authorization | Bearer Token |
| Content-Type | multipart/form-data |

---

## Request Body

| Parameter | Type | Description |
|-----------|------|-------------|
| file | CSV | Healthcare case file |

Example File

cases_import.csv

---

## Success Response

Status Code

200 OK

```json
{
    "totalRecords": 10,
    "successfulImports": 10,
    "failedImports": 0
}
```

---

## Validation Points

- Status Code is 200
- File uploads successfully
- Total records imported are correct
- Imported cases appear in the system
- Invalid rows are reported correctly

---

## Common Error Responses

| Status | Description |
|---------|-------------|
| 400 | Invalid file |
| 401 | Unauthorized |
| 413 | File too large |
| 415 | Unsupported file type |
| 500 | Internal server error |

# Update Case

## Purpose

Update an existing healthcare case.

This endpoint allows modifications to existing case information such as medication details, adverse reactions, case status, or important dates.

---

## Endpoint

PUT /api/cases/{caseId}

Example

PUT /api/cases/HCMS-10001

---

## Request Headers

| Header | Value |
|---------|-------|
| Authorization | Bearer Token |
| Content-Type | application/json |

---

## Request Body

```json
{
    "medication": "Ibuprofen",
    "reaction": "No Reaction",
    "status": "Closed"
}
```

---

## Success Response

Status Code

200 OK

```json
{
    "message": "Case updated successfully"
}
```

---

## Validation Points

- Status Code is 200
- Updated medication is saved
- Updated reaction is saved
- Updated status is reflected
- No unintended fields are modified
- Response body matches expected format

---

## Common Error Responses

| Status | Description |
|---------|-------------|
| 400 | Validation failed |
| 401 | Unauthorized |
| 404 | Case not found |
| 409 | Update conflict |
| 500 | Internal server error |

# Update Case

## Business Purpose
Update an existing healthcare case.

---

## Endpoint
PUT /api/cases/{caseId}

---

## Request Body
```json
{
  "medication": "Ibuprofen",
  "reaction": "None",
  "status": "Closed"
}
```

---

## Success Response
200 OK

```json
{
  "message": "Case updated successfully"
}
```

---

## Error Responses
- 400 Validation error
- 401 Unauthorized
- 404 Case not found
- 409 Conflict
- 500 Server error

---

## Validation Points
- Status code is correct
- Only updated fields are modified
- No unintended data changes
- Response message is correct

---

## Test Scenarios

### Positive
- Valid update request

### Negative
- Invalid case ID
- Missing fields
- Unauthorized request

### Boundary
- Empty values
- Maximum field length values

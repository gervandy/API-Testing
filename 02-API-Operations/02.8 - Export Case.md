# Export Case

## Business Purpose
Export a healthcare case as a CSV file for reporting and validation.

---

## Endpoint
GET /api/cases/export/{caseId}

---

## Path Parameters
| Parameter | Type |
|----------|------|
| caseId | String |

Example: HCMS-10001

---

## Success Response
200 OK

Response: CSV file download

Example: HCMS-10001.csv

---

## Error Responses
- 401 Unauthorized
- 404 Case not found
- 500 Export failed

---

## Validation Points
- Status code is correct
- File downloads successfully
- File format is CSV
- Data matches case details
- No duplicate medication records (HCMS-001 validation)
- Column structure is correct

---

## Test Scenarios

### Positive
- Export valid case successfully

### Negative
- Export invalid case ID
- Export without authentication
- Export deleted case

### Boundary
- Large dataset export
- Long text fields in case data

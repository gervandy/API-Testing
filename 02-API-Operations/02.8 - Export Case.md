# Export Case

## Purpose

Export healthcare case information into a CSV file for reporting, sharing, or regulatory submission.

This endpoint is also used to verify the resolution of Ticket HCMS-001, where duplicate medication records were previously exported.

---

## Endpoint

GET /api/cases/export/{caseId}

Example

GET /api/cases/export/HCMS-10001

---

## Request Headers

| Header | Value |
|---------|-------|
| Authorization | Bearer Token |

---

## Success Response

Status Code

200 OK

Response Type

CSV File Download

Example File

HCMS-10001.csv

---

## Validation Points

- Status Code is 200
- CSV downloads successfully
- Correct Case ID exported
- Patient information is correct
- Medication information is complete
- No duplicate medication records exist
- CSV format matches expected template

---

## Common Error Responses

| Status | Description |
|---------|-------------|
| 401 | Unauthorized |
| 404 | Case not found |
| 500 | Export failed |

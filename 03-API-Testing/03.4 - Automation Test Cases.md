# Automation Test Cases

## Purpose
Defines API test cases selected for automation to support regression testing.

---

## Automation Scope

### Core Regression APIs
- Authentication
- Create Case
- Get Case List
- Get Case
- Update Case
- Delete Case
- Export Case

---

## Automated Test Suite

### Authentication
- TC-A01: Valid login
- TC-A02: Invalid credentials
- TC-A03: Missing credentials

---

### Create Case
- TC-A04: Create valid case
- TC-A05: Missing required field validation
- TC-A06: Duplicate case handling

---

### Get Case List
- TC-A07: Retrieve all cases
- TC-A08: Empty dataset handling

---

### Get Case
- TC-A09: Valid case retrieval
- TC-A10: Invalid case ID

---

### Update Case
- TC-A11: Valid update
- TC-A12: Update non-existent case

---

### Delete Case
- TC-A13: Delete valid case
- TC-A14: Delete already deleted case

---

### Export Case
- TC-A15: Export valid case
- TC-A16: Unauthorized export request

---

## Automation Criteria

- Stable APIs only
- No file-dependent instability (except controlled import/export)
- Repeatable test results
- Independent test execution

---

## Automation Goal
Ensure regression stability of core API workflows after each deployment.
